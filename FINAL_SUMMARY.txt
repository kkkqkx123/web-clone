🔍 问题深层原因
─────────────────────────────────────────────────────────────────────────────

快照中 Vue.js/Nuxt 应用控件无法交互的根本原因：

  Nuxt SSR 应用在快照时被冻结在"初始化中间状态"
  ├─ HTML 被正确捕获 ✓
  ├─ 初始状态被保留 ✓
  └─ Vue 客户端应用未挂载 ✗
     └─ 事件监听器无法绑定
        └─ 所有交互功能失效

关键发现：
  • appHasVueInstance: false      (Vue 实例未挂载)
  • window.Vue: undefined          (Vue 库未初始化)
  • 手动 click 事件: 无反应         (无事件监听器)
  • 应用状态: 冻结                  (处于 SSR 中间状态)


✅ 实施的修复
─────────────────────────────────────────────────────────────────────────────

1️⃣ 改进 Playwright 等待机制
   文件: src/adapters/automation/playwright/adapter.ts
   
   • 检测 Nuxt SSR 应用特征 (window.__NUXT__)
   • 等待 Vue 组件真正挂载 (#__nuxt__.__vue__)
   • 增加 1 秒额外延迟确保事件绑定完成
   • 添加调试日志用于诊断
   
   效果: Playwright 现在等待应用真正初始化，而非仅网络空闲

2️⃣ 注入 Vue 初始化脚本
   文件: src/assembler.ts
   
   • 在生成的快照中注入 Vue hydration 脚本
   • 脚本自动检测未初始化的 Vue 应用
   • 尝试重新触发 Nuxt/Vue 挂载
   • 最多重试 20 次，500ms 间隔
   
   效果: 快照打开时自动重新激活 Vue 应用

3️⃣ 增强资源验证
   文件: src/validators.ts
   
   • 新增 JavaScript 内容检查函数
   • 检测 HTML 伪装的 JavaScript
   • 改进脚本验证逻辑
   
   效果: 立即检测并拒绝无效脚本

4️⃣ 改进重定向处理
   文件: src/fetcher.ts
   
   • 追踪完整的重定向链
   • 检测重定向后的内容类型错误
   • 自动重试
   
   效果: 诊断和恢复重定向问题
