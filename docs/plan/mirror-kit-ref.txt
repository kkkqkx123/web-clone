基于对 web-mirror (当前项目) 和 web-clone (目标对比项目) 的深度分析，以下是 web-clone 需要补充的功能总结文档。
web-clone 功能缺口分析与补充建议
1. 核心结论
web-clone 在动态内容获取（SPA、认证、浏览器自动化）方面具有绝对优势，但在资源发现的深度、特定场景的针对性处理以及离线维护工具链上不如 web-mirror 完善。
若要使 web-clone 成为一个全能的网站镜像工具，需重点补充以下三大类功能：
深层资源递归挖掘机制（弥补单纯依赖 DOM 解析的不足）
专用场景增强模块（CMS 媒体、视频流、隐藏接口）
离线校验与维护工具集（坏缓存清理、完整性验证）
2. 详细功能缺口分析
2.1 资源发现策略：从“静态扫描”到“深度递归”
现状差异：
web-mirror: 采用多轮递归扫描（最多4轮）。它不仅解析 HTML，还下载并解析 JS/CSS/JSON 文件，从中提取下一层的资源 URL。这能发现隐藏在代码逻辑中的图片、字体或 API 地址。
web-clone: 主要依赖浏览器渲染后的 DOM 树 和静态正则匹配。如果资源是通过 JS 动态拼接 URL 或在非标准 JSON 结构中，极易漏抓。
🔴 需补充功能：
多轮次资源提取引擎：
实现类似 mirror-assets.js 的逻辑：下载 JS/CSS/JSON -> 正则提取 URL -> 加入下载队列 -> 再次扫描新下载的文件。
支持配置递归深度（如 depth: 4）和并发数。
JS/CSS 内部链接重写与提取：
不仅替换 HTML 中的链接，还需解析 JS 文件中的 url(), src=, fetch('...') 等模式，提取潜在资源。
JSON 深度扫描器：
针对 CMS 返回的复杂嵌套 JSON 数据，实施递归遍历，提取所有符合媒体格式的字符串值。
2.2 特定场景增强：CMS 与多媒体
现状差异：
web-mirror: 拥有专门的 mirror-cms-media.js 和 find-video-refs.js。它能识别 WordPress/Drupal 等 CMS 的特定 JSON 结构，甚至能推断缓存 ID，专门处理 .m3u8, .mp4 等流媒体。
web-clone: 通用性强但缺乏针对性，容易遗漏非标准标签引用的媒体资源。
🔴 需补充功能：
CMS 媒体专用抓取器：
增加针对常见 CMS (WordPress, Contentful, Strapi) 的 JSON 路径识别规则。
实现“缓存 ID 推断”逻辑，批量下载未直接链接但存在于数据库配置中的图片。
流媒体与视频增强处理：
集成 find-video-refs 逻辑，专门扫描 <script> 标签内的视频配置对象。
支持 HLS (.m3u8) 和 DASH (.mpd) 播放列表的解析与分片下载（目前可能仅下载主文件）。
WASM 与大二进制文件支持：
显式增加对 .wasm, .bin, .exe 等二进制文件的识别和完整下载策略，防止被误判为文本截断。
2.3 质量控制与维护工具链
现状差异：
web-mirror: 提供完整的事后工具集：validate-assets.js (校验文件头/格式), 坏缓存清理逻辑，以及详细的错误报告。
web-clone: 侧重于“一次性获取”，缺乏对已下载内容的长期维护和完整性校验工具。
🔴 需补充功能：
离线资产验证器 (Validator)：
开发独立脚本，扫描输出目录：
检查图片/视频文件头是否损坏（Magic Number 校验）。
检查 JSON 文件是否可解析。
检测 HTML 中是否残留未替换的外部链接。
坏缓存清理与重试机制：
识别大小为 0 或格式错误的文件，自动标记并触发重试下载。
提供 --clean 参数，一键清理无效缓存。
资源去重与统计报告：
生成详细的下载报告：成功/失败数量、节省流量估算、遗漏资源列表。
2.4 架构灵活性：混合模式
现状差异：
web-mirror: 纯 Node.js HTTP 请求，速度极快，适合大规模静态资源。
web-clone: 强依赖 Headless Browser，速度慢，资源消耗大。
🔴 需补充功能：
混合抓取策略 (Hybrid Mode)：
首屏用浏览器：使用 Playwright 获取 SPA 渲染后的 HTML 和关键资源。
深层用 HTTP：提取出的静态资源列表（图片、CSS、JS），脱离浏览器，使用原生 http/https 模块高速并发下载。
价值：兼顾 SPA 支持能力和大规模下载的速度。

4. 总结
web-clone 目前是一个优秀的**“前端渲染快照工具”，但还不是一个成熟的“全站镜像系统”**。
通过吸收 web-mirror 在深度资源挖掘、专用场景处理和质量保障体系方面的优势，web-clone 将从单纯的“页面截图/HTML 保存”进化为能够应对企业级复杂网站（含 SPA + CMS + 多媒体）的全功能离线镜像解决方案。
