## 核心结论

**不要使用 Tree-sitter，不要从零编写 AST 解析器。**

**最佳方案：分层组合策略**
- **HTML/CSS** → 使用成熟的专用解析器（htmlparser2 + postcss）
- **JavaScript** → 使用 Babel（AST 操作）
- **整体协调** → 用 Node.js 脚本编排

## 技术选型对比分析

### 1️⃣ HTML 解析方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **cheerio** | jQuery 风格 API，上手快，容错强 | 不生成完整 AST，难以深度修改 | ⭐⭐⭐⭐⭐ **强烈推荐** |
| **htmlparser2** | 底层解析器，性能极高，流式处理 | API 较底层，需要自己遍历 | ⭐⭐⭐⭐ 推荐（cheerio 底层就是它） |
| **parse5** | 严格遵循 HTML5 规范，输出标准 AST | 过度严谨，对不规范的 HTML 支持差 | ⭐⭐⭐ 特定场景 |
| **Tree-sitter-html** | 增量解析，错误恢复好 | 社区不活跃，生态不完善 | ⭐ 不推荐 |

**实战建议**：
```javascript
// ✅ 最佳实践：cheerio 就够了
const cheerio = require('cheerio');
const $ = cheerio.load(htmlString, {
  // 对不规范 HTML 更宽容
  xmlMode: false,
  decodeEntities: true
});

// 丰富的选择器和遍历能力
$('[data-component]').each((i, el) => {
  const componentName = $(el).attr('data-component');
  // 提取模板、样式...
});
```

| 方案 | 优点 | 缺点 | 推荐度 |

```javascript
// ✅ 最佳实践：PostCSS + 自定义插件
const postcss = require('postcss');
const cssAST = postcss.parse(cssString);

// 遍历 CSS 规则
cssAST.walkRules((rule) => {
  // 提取选择器、声明等
  rule.selectors.forEach(selector => {
    // 判断属于哪个组件
  });
});

// 也可以用 postcss 插件提取变量
const variables = {};
cssAST.walkDecls((decl) => {
  if (decl.prop.startsWith('--')) {
    variables[decl.prop] = decl.value;
  }
});
```

## 关于 Tree-sitter 的深度分析

### 为什么不推荐 Tree-sitter？

虽然 Tree-sitter 在某些场景很优秀，但**不适合这个任务**：

| 问题 | 说明 |
|------|------|
| **AST 标准不统一** | Tree-sitter 的 AST 结构是自定义的，不遵循 ESTree 等标准 |
| **工具链不成熟** | 修改、生成代码的能力远不如 Babel |
| **社区生态弱** | 中文文档少，遇问题难以解决 |
| **不适合批处理** | 它擅长 IDE 增量解析，而非一次性批量转换 |
| **学习成本高** | 需要同时学多个语言的语法定义 |

### Tree-sitter 的唯一优势场景

```javascript
// ❌ 不要用 Tree-sitter 做这个：
// "解析整个项目，生成组件清单"

// ✅ 应该用 Tree-sitter 做这个：
// "在编辑器中实时解析，提供语法高亮和错误提示"
```

### 核心流程编排

```javascript
// index.js - 主入口
const { analyzeHTML } = require('./parsers/html-analyzer');
const { analyzeCSS } = require('./parsers/css-analyzer');
const { analyzeJS } = require('./parsers/js-analyzer');
const { generateOutput } = require('./generators/output-generator');

class BundleToComponentConverter {
  constructor(html, css, js) {
    this.html = html;
    this.css = css;
    this.js = js;
  }
  
  async convert() {
    // 1. 并行解析三种语言
    const [htmlResult, cssResult, jsResult] = await Promise.all([
      analyzeHTML(this.html),
      analyzeCSS(this.css),
      analyzeJS(this.js)
    ]);
    
    // 2. 关联分析（关键步骤）
    const componentMap = this.correlate(htmlResult, cssResult, jsResult);
    
    // 3. 生成组件化输出
    return generateOutput(componentMap);
  }
  
  correlate(html, css, js) {
    // 核心关联逻辑
    // 1. 根据 HTML 结构识别组件边界
    // 2. 匹配对应的 CSS 规则（基于 class/ID）
    // 3. 关联 JS 操作（基于 DOM 引用）
    return {
      components: this.extractComponents(html, css, js),
      globalStyles: this.extractGlobalStyles(css),
      sharedLogic: this.extractSharedLogic(js)
    };
  }
}
```

### HTML 分析器（使用 cheerio）

```javascript
// parsers/html-analyzer.js
const cheerio = require('cheerio');

function analyzeHTML(htmlString) {
  const $ = cheerio.load(htmlString);
  const components = [];
  
  // 1. 识别组件根节点
  const componentRoots = findComponentRoots($);
  
  componentRoots.forEach((root, index) => {
    const component = {
      id: root.attribs['data-component-id'] || `comp-${index}`,
      name: root.attribs['data-component'] || inferComponentName(root),
      template: $.html(root),
      // 记录动态点
      bindings: extractBindings($, root),
      events: extractEvents($, root),
      conditions: extractConditions($, root),
      // DOM 引用（用于 JS 关联）
      refs: extractRefs($, root)
    };
    components.push(component);
  });
  
  return {
    components,
    // 保留未被组件包裹的根级结构
    rootTemplate: $.html('body').children().not('[data-component]')
  };
}

function findComponentRoots($) {
  // 策略1: 显式标记
  const marked = $('[data-component]').toArray();
  if (marked.length) return marked;
  
  // 策略2: 语义标签
  const semantic = $('header, footer, nav, main, section, article').toArray();
  if (semantic.length) return semantic;
  
  // 策略3: 深度切分（超过4层强制拆分）
  return splitByDepth($);
}

function extractBindings($, element) {
  const bindings = [];
  $(element).find('[data-binding]').each((i, el) => {
    bindings.push({
      selector: $(el).attr('data-binding'),
      element: $(el).prop('tagName'),
      text: $(el).text().trim(),
      // 猜测绑定类型
      type: guessBindingType($(el))
    });
  });
  return bindings;
}
```

### CSS 分析器（使用 PostCSS）

```javascript
// parsers/css-analyzer.js
const postcss = require('postcss');
const selectorParser = require('postcss-selector-parser');

function analyzeCSS(cssString) {
  const root = postcss.parse(cssString);
  const result = {
    variables: {},
    globalRules: [],
    componentRules: {},
    dynamicStyles: []
  };
  
  // 1. 提取 CSS 变量
  root.walkDecls((decl) => {
    if (decl.prop.startsWith('--')) {
      result.variables[decl.prop] = decl.value;
    }
  });
  
  // 2. 按组件归类规则
  root.walkRules((rule) => {
    const componentName = inferComponentFromSelector(rule.selector);
    if (componentName) {
      if (!result.componentRules[componentName]) {
        result.componentRules[componentName] = [];
      }
      result.componentRules[componentName].push(rule.toString());
    } else {
      result.globalRules.push(rule.toString());
    }
    
    // 3. 识别动态样式（可能被 JS 修改）
    if (isDynamicStyle(rule)) {
      result.dynamicStyles.push({
        selector: rule.selector,
        declarations: extractDynamicProps(rule),
        // 记录原代码中的修改方式
        originalJS: findJSModifications(rule.selector)
      });
    }
  });
  
  return result;
}

function inferComponentFromSelector(selector) {
  // 例如: .header__logo → Header 组件
  // .counter__display → Counter 组件
  const match = selector.match(/\.([a-zA-Z]+)__/);
  return match ? match[1] : null;
}
```

### JS 分析器（使用 Babel）

```javascript
// parsers/js-analyzer.js
const parser = require('@babel/parser');
const traverse = require('@babel/traverse').default;
const t = require('@babel/types');

function analyzeJS(jsString) {
  const ast = parser.parse(jsString, {
    sourceType: 'unambiguous',
    errorRecovery: true // 即使有语法错误也尽量解析
  });
  
  const result = {
    state: [],
    methods: [],
    events: [],
    domRefs: [],
    lifecycles: {
      mount: null,
      unmount: null,
      updates: []
    },
    dependencies: [],
    // 无法自动转换的部分
    manualTodos: []
  };
  
  traverse(ast, {
    // 1. 提取状态变量
    VariableDeclarator(path) {
      if (isStateVariable(path)) {
        result.state.push({
          name: path.node.id.name,
          initialValue: extractValue(path.node.init),
          scope: inferScope(path)
        });
      }
    },
    
    // 2. 提取方法
    FunctionDeclaration(path) {
      if (isHandlerFunction(path)) {
        result.methods.push({
          name: path.node.id.name,
          params: path.node.params.map(p => p.name),
          body: path.node.body,
          // 提取 DOM 操作
          domOperations: extractDOMOps(path)
        });
      }
    },
    
    // 3. 提取事件监听
    CallExpression(path) {
      if (isAddEventListener(path)) {
        result.events.push({
          target: path.node.arguments[0].value,
          event: path.node.arguments[1].value,
          handler: path.node.arguments[2].name,
          // 推测对应的 DOM 元素
          element: inferTargetElement(path)
        });
      }
      
      // 4. 识别 DOMContentLoaded
      if (isDOMContentLoaded(path)) {
        result.lifecycles.mount = extractCallback(path);
      }
      
      // 5. 识别定时器（需要清理）
      if (isTimer(path)) {
        result.lifecycles.updates.push({
          type: path.node.callee.name,
          interval: path.node.arguments[0],
          // 需要清理标记
          needsCleanup: true
        });
      }
    },
    
    // 6. 识别 DOM 查询（需要转为 ref）
    CallExpression(path) {
      if (path.node.callee.property?.name === 'getElementById' ||
          path.node.callee.property?.name === 'querySelector') {
        result.domRefs.push({
          selector: path.node.arguments[0].value,
          variable: findParentVariable(path),
          // 标记需要转为 ref
          needsRef: true
        });
        // 添加 TODO 注释
        result.manualTodos.push({
          description: `将 DOM 查询 '${path.node.arguments[0].value}' 转为 ref`,
          location: path.node.loc
        });
      }
    }
  });
  
  return result;
}

function isStateVariable(path) {
  // 启发式判断：被修改且绑定到 DOM 的变量
  const binding = path.scope.getBinding(path.node.id.name);
  return binding.references.some(ref => {
    // 检查是否被修改
    return ref.path.isAssignmentExpression() ||
           // 检查是否用于 DOM 更新
           ref.path.parent?.property?.name === 'innerText' ||
           ref.path.parent?.property?.name === 'textContent';
  });
}
```

## 进阶：关联分析（核心难点）

```javascript
// correlation-engine.js
class CorrelationEngine {
  constructor(htmlResult, cssResult, jsResult) {
    this.html = htmlResult;
    this.css = cssResult;
    this.js = jsResult;
  }
  
  correlate() {
    const componentMap = new Map();
    
    // 1. 用 HTML 组件作为主键
    this.html.components.forEach(comp => {
      const component = {
        name: comp.name,
        template: comp.template,
        bindings: comp.bindings,
        events: comp.events,
        conditions: comp.conditions,
        // 待填充
        styles: [],
        logic: []
      };
      
      // 2. 关联 CSS
      component.styles = this.matchStyles(comp);
      
      // 3. 关联 JS
      component.logic = this.matchLogic(comp);
      
      // 4. 标注未匹配的部分
      component.todos = this.findUnmatched(comp);
      
      componentMap.set(comp.name, component);
    });
    
    return componentMap;
  }
  
  matchStyles(component) {
    // 通过 class 名称匹配
    const classNames = extractClassNames(component.template);
    const matched = [];
    
    classNames.forEach(cls => {
      const rules = this.css.componentRules[component.name] || [];
      const matchedRules = rules.filter(rule => 
        rule.includes(`.${cls}`)
      );
      matched.push(...matchedRules);
    });
    
    return matched;
  }
  
  matchLogic(component) {
    const logic = [];
    
    // 通过 DOM 引用匹配
    component.bindings.forEach(binding => {
      // 在 JS 中查找操作此元素的代码
      const jsOps = this.js.domRefs.filter(ref => 
        ref.selector.includes(binding.selector) ||
        binding.selector.includes(ref.selector)
      );
      logic.push(...jsOps);
    });
    
    // 通过事件匹配
    component.events.forEach(event => {
      const jsEvents = this.js.events.filter(jsEvent =>
        jsEvent.event === event.type &&
        (jsEvent.target === event.target || 
         jsEvent.element === event.target)
      );
      logic.push(...jsEvents);
    });
    
    return logic;
  }
}
```

## 输出生成器

```javascript
// generators/output-generator.js
const fs = require('fs-extra');
const path = require('path');
const prettier = require('prettier');

function generateOutput(componentMap, globalStyles, sharedLogic) {
  const outputDir = './output';
  fs.emptyDirSync(outputDir);
  
  // 1. 生成组件文件
  componentMap.forEach((component, name) => {
    const componentDir = path.join(outputDir, 'components', name);
    fs.ensureDirSync(componentDir);
    
    // 写入模板
    fs.writeFileSync(
      path.join(componentDir, 'template.html'),
      prettier.format(component.template, { parser: 'html' })
    );
    
    // 写入样式
    fs.writeFileSync(
      path.join(componentDir, 'style.css'),
      prettier.format(component.styles.join('\n'), { parser: 'css' })
    );
    
    // 写入清单
    fs.writeFileSync(
      path.join(componentDir, 'manifest.json'),
      JSON.stringify({
        name: component.name,
        bindings: component.bindings,
        events: component.events,
        conditions: component.conditions,
        todos: component.todos,
        dependencies: {
          styles: component.styleDependencies || [],
          logic: component.logicDependencies || []
        }
      }, null, 2)
    );
    
    // 保留原始逻辑（供参考）
    if (component.originalLogic) {
      fs.writeFileSync(
        path.join(componentDir, 'logic.original.js'),
        component.originalLogic
      );
    }
  });
  
  // 2. 生成全局样式
  fs.writeFileSync(
    path.join(outputDir, 'styles', 'variables.css'),
    prettier.format(globalStyles.variables, { parser: 'css' })
  );
  
  // 3. 生成索引文件
  generateIndex(componentMap, outputDir);
  
  // 4. 生成 README
  generateREADME(componentMap, outputDir);
}
```

## 总结：技术选型决策表

| 语言 | 推荐工具 | 替代方案 | 是否考虑 Tree-sitter |
|------|---------|---------|---------------------|
| **HTML** | **cheerio** | parse5 | ❌ 不推荐 |
| **CSS** | **postcss** | css-tree | ❌ 不推荐 |
| **JavaScript** | **@babel/parser + traverse** | acorn | ❌ 不推荐 |
| **整体编排** | **Node.js + 自定义脚本** | - | ❌ 不推荐 |

---

## 最终建议

1. **用对工具**：cheerio + PostCSS + Babel 是黄金组合
2. **分层设计**：解析层 → 关联层 → 生成层，职责清晰
3. **容错优先**：原代码可能不规范，用 `errorRecovery: true`
4. **保留注释**：无法自动转换的部分用 `/* TODO */` 标记
5. **渐进迭代**：先处理 80% 的简单情况，再处理 20% 的复杂情况

> **记住**：你的目标是生成**人工可用的半成品**，而不是完美的自动化工具。选择最成熟的工具链，把精力花在关联逻辑和输出质量上，而不是底层解析实现。
